# Go 練習筆記 Skill

## 工作流程

### 判斷模式

- **模式 A（主題生成）**：使用者描述主題（如「Go 的 slice」），無現有程式碼 → 自行生成範例程式碼再整理
- **模式 B（程式碼整理）**：使用者提供現有 `main.go` 或指定路徑 → 讀取後整理

### 執行步驟

1. **確認目標路徑**：詢問或推斷要存放的目錄（參考 CLAUDE.md 的目錄結構）
2. **產生/整理 `main.go`**：加上繁體中文區塊註解
3. **建立 `NOTE.md`**：在同一目錄下建立 Markdown 筆記
4. **確認內容完整**：概念說明、程式碼範例、常見錯誤三項都要涵蓋

格式規範與範例見 `references/note-format.md`。

## 注意事項

- 所有說明使用**繁體中文**
- `main.go` 區塊概念用 `// ===== 標題 =====`，行內說明用 `// 說明`
- 程式碼範例需能獨立執行（完整的 `package main` + `import`）
- `NOTE.md` 放在與 `main.go` 相同目錄
